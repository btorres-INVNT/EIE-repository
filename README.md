A task or feature is considered complete when all of the following criteria are met:
- Code is written and follows team coding standards
- Unit tests are written and passing
- A pull request is opened
- The pull request is reviewed and approved by at least one team member
- The pull request is merged into the main branch
- The feature is verified in the staging environment
- The related issue is closed

  *This definition of done will be reviewed and refined with the team during retrospectives.*
